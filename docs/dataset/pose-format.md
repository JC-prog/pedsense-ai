# Pose Format

The pose dataset is generated by running a pretrained YOLO-Pose model on extracted JAAD frames. It contains pedestrian bounding boxes and 17 COCO body keypoints per detection.

## Directory Structure

```
data/processed/pose/
  images/
    train/
      video_0001_frame_000030.jpg
      ...
    val/
      ...
  labels/
    train/
      video_0001_frame_000030.txt
      ...
    val/
      ...
  data.yaml
```

## Label Format

Each `.txt` file corresponds to one frame. Each line is one detected pedestrian:

```
0 cx cy w h  kp1x kp1y 2  kp2x kp2y 2  ...  kp17x kp17y 2
```

| Field | Description |
|-------|-------------|
| `0` | Class ID (always 0 = pedestrian) |
| `cx cy` | Bounding box center (normalized 0–1) |
| `w h` | Bounding box width/height (normalized 0–1) |
| `kpNx kpNy` | Keypoint N coordinates (normalized 0–1) |
| `2` | Visibility flag: `2` = visible |

### COCO Keypoint Order (17 points)

| Index | Keypoint |
|-------|----------|
| 0 | Nose |
| 1 | Left eye |
| 2 | Right eye |
| 3 | Left ear |
| 4 | Right ear |
| 5 | Left shoulder |
| 6 | Right shoulder |
| 7 | Left elbow |
| 8 | Right elbow |
| 9 | Left wrist |
| 10 | Right wrist |
| 11 | Left hip |
| 12 | Right hip |
| 13 | Left knee |
| 14 | Right knee |
| 15 | Left ankle |
| 16 | Right ankle |

## data.yaml

```yaml
path: /absolute/path/to/data/processed/pose
train: images/train
val: images/val
nc: 1
names:
  - pedestrian
kpt_shape:
  - 17
  - 3
```

## Generation

```bash
# Extract frames first
uv run pedsense preprocess frames

# Generate pose labels (downloads yolo11n-pose.pt on first run)
uv run pedsense preprocess pose

# Options
uv run pedsense preprocess pose --pose-variant yolo11m-pose --conf 0.4
```

See [`pedsense preprocess`](../cli/preprocess.md) for full options.

## Usage with Ultralytics

The output is ready to train a custom YOLO-Pose model:

```python
from ultralytics import YOLO

model = YOLO("yolo11n-pose.pt")
model.train(data="data/processed/pose/data.yaml", epochs=100)
```

## Notes

- Frames with no detected pedestrians have no label file (YOLO convention for empty frames)
- The 80/20 train/val split is video-level (same seed as YOLO and ResNet steps)
- Source images are JAAD dashcam footage at 1920×1080
